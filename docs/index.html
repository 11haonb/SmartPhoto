核心功能

四大 AI 能力，全自动处理

从照片导入到最终输出，SmartPhoto 覆盖整个照片管理生命周期

📅
时间线自动整理
自动提取 EXIF 元数据，按拍摄日期、地点、相机型号智能分组，无需手动建文件夹。
EXIF 提取 GPS 定位 自动分组
🔍
AI 质量检测
检测模糊（Laplacian 方差）、过曝、欠曝、截图等问题，将低质量照片标记为可删除。
模糊检测 曝光分析 截图识别
🤖
智能场景分类
识别人像（自拍/群照/肖像）、风景（自然/建筑/城市）、美食、文档等场景类别，精确打标。
人像识别 场景分类 置信度评分
⭐
最佳照片挑选
用感知哈希（pHash）找出相似/重复照片组，AI 自动从每组中挑出构图最佳、画质最清晰的一张。
pHash 去重 自动精选 分组对比
处理流程

五阶段 AI 流水线

每次处理任务都经过完整的五个阶段，通过 Celery 异步队列调度，前端实时轮询进度。

1
EXIF 提取与时间线
读取拍摄时间、相机型号、GPS 坐标和画面方向，按日期自动分组建立时间线。
taken_at camera_model GPS pHash 计算
2
质量分析
Laplacian 方差检测模糊，直方图分析曝光，分辨率模式识别截图。
模糊 <100 过曝 >230 欠曝 <30
3
智能分类
AI 引擎识别主类别与子类别，输出置信度评分，写入 photo_analyses 表。
person landscape food document
4
相似度分组
比较感知哈希汉明距离，≤10 判定为相似，统一分配 group_id。
pHash 对比 汉明距离≤10 group_id
5
最佳照片挑选
AI 对每个相似组内的照片排名，标记 is_best_in_group = true，完成任务。
AI 排名 is_best 任务完成

快速开始

三步完成本地部署

基于 Docker Compose，开发环境一键拉起，包含全部依赖服务

1
克隆仓库
克隆项目，复制环境变量配置文件
# 克隆项目 git clone https://github.com/your/smartphoto cd smartphoto/photo-organizer-backend # 复制并编辑配置 cp .env.example .env
2
启动服务
Docker Compose 自动拉起 API、Worker、PostgreSQL、Redis、MinIO
# 构建并启动全部服务 docker compose \ -f docker-compose.dev.yml \ up --build -d # 查看服务状态 docker compose ps
3
访问 & 测试
API 文档在 28000 端口，Web Demo 在 3000 端口，MinIO 控制台在 29001 端口
# API 文档 (Swagger) http://localhost:28000/docs # Web Demo http://localhost:3000 # MinIO 控制台 http://localhost:29001
